# 任務圖片資料夾設置指南

## 📁 資料夾結構

請在 `public/` 目錄下創建以下資料夾結構：

```
public/
├── images/
│   ├── gifts/          ✅ 禮品圖片（已有）
│   │   ├── gift-1.jpg
│   │   ├── gift-2.jpg
│   │   └── ... (共32張)
│   │
│   └── tasks/          🆕 任務圖片（需要創建）
│       ├── task-1.jpg
│       ├── task-2.jpg
│       ├── task-3.jpg
│       └── ... (共30張)
```

## 🎯 圖片命名邏輯

| 7-12 | 理財學習任務 | task-7.jpg ~ task-12.jpg | 💰 |

## 📐 推薦圖片規格

為了與任務卡片設計相符，建議：

- **尺寸**：1000 x 1000px (1:1 正方形比例)
- **格式**：JPG (高品質壓縮)
- **檔案大小**：100-200KB (建議優化)

## 🎨 設計建議

每張任務圖片可考慮包含：

1. **背景**：與任務分類相符的色系
   - 日常互動：藍色系 (#2563EB)
   - 理財學習：綠色系 (#10B981)
   - 投資實踐：紫色系 (#7C3AED)
   - 永續行動：翠綠色系 (#059669)
   - 社群成就：橙色系 (#F97316)

2. **視覺元素**：任務的代表圖示或場景圖片
3. **文字**：任務標題（選填）

## 🔗 代碼引用位置

所有任務已經配置好圖片路徑，圖片將自動在以下位置顯示：

- **任務詳細模態框** - 左側大型圖片區域 (192 x 192px)
- 代碼位置：`src/components/modals/TaskDetailModal.vue` 第 22-26 行

## 💡 額外說明

### 為什麼圖片放在 `public/` 而不是 `src/`？

1. **性能優化**：
   - `public/` 中的資源不經過 webpack 打包
   - 減少最終打包體積
   - 更快的加載速度

2. **靜態資產管理**：
   - 直接通過 HTTP 請求加載
   - 修改圖片無需重新構建項目
   - 便於團隊協作編輯

3. **路徑簡化**：
   - 使用簡潔的相對路徑：`/images/tasks/task-1.jpg`
   - 無需在代碼中導入

### 如何更換圖片？

只需替換 `public/images/tasks/` 資料夾中對應編號的圖片檔案，無需修改代碼。

## 開發動機

為企業客戶設計的**積分獎勵與分級制度平台**。

透過完成任務累積點數並兌換禮品，提升客戶互動與長期黏著度。

**目標用戶**：金融機構、銀行、證券公司的會員經營系統

## 雙軌積分制度

**核心設計亮點**

| 積分類型 | 用途 | 特性 |
|----------|------|------|
| 升級積分 (levelPoints) | 累積升級會員等級 | 只增不減 |
| 兌換積分 (rewardPoints) | 兌換禮品使用 | 可消費扣除 |

### 會員等級系統

| 等級 | 名稱 | 升級門檻 | 積分加成 |
|------|------|----------|----------|
| Lv1 | Explorer 探索者 | 0 點 | 1.0x |
| Lv2 | Creator 創造者 | 250 點 | 1.15x |
| Lv3 | Visionary 先行者 | 750 點 | 1.3x |
| Lv4 | Luminary 閃耀者 | 1500 點 | 1.5x |

**設計理念**：等級越高，每次完成任務獲得的積分越多，形成正向循環。

## 任務亮點（重點展開）

### Task 1：每日登入
- **功能**：用戶每日登入即可獲得積分
- **後端實現**：
  - 記錄至 `activity_logs` 表（userId, loginTime, IP）
  - 計算連續登入天數
  - 更新 `user_levels` 積分

### Task 10：建立投資組合
- **功能**：用戶選擇 3-5 檔基金，分配虛擬資金比例
- **後端實現**：
  - 存入 `investment_portfolios` 表
  - 定期計算虛擬淨值變化
  - 可串接 TWSE API 取得即時行情（加分項）

### Task 16：記錄虛擬交易
- **功能**：用戶輸入股票代號、數量、買入價格
- **後端實現**：
  - 計算損益 = (現價 - 買入價) × 數量
  - 存入 `trading_records` 表
  - 更新 `performance_history` 績效統計

## 積分定價策略

**1 積分 ≈ NT$3**

| 設計考量 | 說明 |
|----------|------|
| 避免太便宜 | 1積分=0.5元 → 兌換太快，企業成本爆炸 |
| 避免太貴 | 1積分=5元 → 禮物難換，用戶失去動力 |
| 最佳平衡 | 1積分≈3元 → Lv1 可換小禮，Lv4 可換大獎 |

### 庫存機制（製造稀缺感）

| 狀態 | 數量 | 心理效果 |
|------|------|----------|
| 🟢 充足 | 99+ | 安心兌換 |
| 🟡 適量 | 50-99 | 建議盡早 |
| 🟠 有限 | 10-49 | 先兌先得 |
| 🔴 稀少 | <10 | 收藏價值 |

## JWT 身份驗證

### 什麼是 JWT？
JSON Web Token - 一種無狀態的身份驗證機制

### 運作流程

```
1. 用戶登入 → 提交帳號密碼
2. 後端驗證 → 帳密正確
3. 發送 Token → 後端產生 JWT 回傳
4. 前端儲存 → 存於 localStorage
5. API 請求 → 每次帶上 Token
6. 後端驗證 → 檢查 Token 有效性
```

### JWT 優點

| 優點 | 說明 |
|------|------|
| 無狀態 | 不需 Session，減輕伺服器負擔 |
| 跨域支援 | 適合前後端分離架構 |
| 安全性高 | 可設定過期時間、加密簽名 |
| 擴展性強 | 可攜帶用戶資訊（userId, role） |
